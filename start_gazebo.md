# Запуск Gazebo Harmonic — warehouse_phase0

## Быстрый запуск

### Вариант 1: Скрипт (рекомендуется)

Из **PowerShell** (Windows):

```powershell
wsl -d Ubuntu-22.04 -u imuzolev -- bash /mnt/c/CORTEXIS/Drone_new_method/scripts/launch_sim.sh

wsl -d Ubuntu-22.04 -u imuzolev -- bash /mnt/c/CORTEXIS/Drone_new_method/scripts/start_gazebo.sh
```

Или из **WSL терминала**:

```bash
bash /mnt/c/CORTEXIS/Drone_new_method/scripts/start_gazebo.sh
```

### Вариант 2: Вручную из WSL

```bash
cd /mnt/c/CORTEXIS/Drone_new_method
export GZ_SIM_RESOURCE_PATH="/mnt/c/CORTEXIS/Drone_new_method/simulation/models:$HOME/PX4-Autopilot/Tools/simulation/gz/models"
gz sim simulation/worlds/warehouse_phase0.sdf
```

### Вариант 3: Через PX4 SITL (полная симуляция с дроном)

```bash
cd ~/PX4-Autopilot
PX4_GZ_WORLD=warehouse_phase0 PX4_GZ_MODEL_POSE="-9,0,0.2,0,0,0" make px4_sitl gz_x500_warehouse
```

При запуске через PX4 дрон появится в мире автоматически, а все сенсоры (LiDAR, камера) будут активны.

## Управление камерой в Gazebo

### Мышь


| Действие                      | Что делает                                           |
| ----------------------------- | ---------------------------------------------------- |
| **Правая кнопка + движение**  | Вращение камеры (orbit) вокруг точки фокуса          |
| **Средняя кнопка + движение** | Панорамирование (pan) — сдвиг камеры вбок/вверх/вниз |
| **Колесо прокрутки**          | Приближение / отдаление (zoom)                       |
| **Левый клик на модель**      | Выбрать объект (подсвечивается)                      |
| **Двойной клик на модель**    | Переместить фокус камеры на объект                   |


### Правый клик на объект (контекстное меню)


| Пункт              | Что делает                                     |
| ------------------ | ---------------------------------------------- |
| **Move To**        | Камера плавно подлетает к объекту              |
| **Follow**         | Камера следует за объектом (полезно для дрона) |
| **View → Look At** | Камера смотрит на объект, но не перемещается   |


### Клавиатура


| Клавиша   | Что делает                                          |
| --------- | --------------------------------------------------- |
| **Esc**   | Сбросить текущий инструмент (вернуться к навигации) |
| **T**     | Инструмент трансляции (перемещение объекта)         |
| **R**     | Инструмент вращения                                 |
| **Space** | Пауза / продолжение симуляции                       |


### Полезные действия с камерой

**Переместиться к штрихкодам:**

1. В панели слева (Entity Tree) найти модель `bcL_1_1`
2. Правый клик → **Move To** — камера подлетит к первому штрихкоду левого стеллажа
3. Колесом приблизиться для проверки текстуры

**Следить за дроном (если запущен через PX4):**

1. В Entity Tree найти `x500_warehouse`
2. Правый клик → **Follow**
3. Камера будет автоматически следовать за дроном

**Сбросить ракурс:**

1. Двойной клик на пустое место пола → камера сфокусируется на центр

## GUI: полезные панели

Через меню **Plugins** (иконка ⋮ вверху справа) можно добавить:


| Плагин                  | Зачем                                                          |
| ----------------------- | -------------------------------------------------------------- |
| **Entity Tree**         | Дерево всех объектов — быстрый доступ к любой модели           |
| **Component Inspector** | Просмотр позиции, материалов, физики выбранного объекта        |
| **Image Display**       | Просмотр топика камеры (`/drone/camera/image_raw`) прямо в GUI |
| **Transform Control**   | Перемещение/вращение объектов в реальном времени               |
| **View Angle**          | Быстрое переключение вида: сверху, сбоку, спереди              |

## Troubleshooting

### Окно Gazebo не открывается (WSL2)

Нужен X-сервер или WSLg. Проверить:

```bash
echo $DISPLAY
# Должно быть что-то вроде :0 или :0.0
```

Если пусто — WSLg не активен. Обновить WSL:

```powershell
wsl --update
```

### Текстуры не отображаются (белые прямоугольники)

Проверить что PNG существуют:

```bash
ls /home/imuzolev/gz_barcodes/ | wc -l
# Должно быть 48
```

Проверить путь в SDF:

```bash
grep albedo_map /mnt/c/CORTEXIS/Drone_new_method/simulation/worlds/warehouse_phase0.sdf | head -1
# Должен указывать на /home/imuzolev/gz_barcodes/barcode_L_01_1.png
```

### Gazebo запускается, но мир пустой

Проверить `GZ_SIM_RESOURCE_PATH`:

```bash
echo $GZ_SIM_RESOURCE_PATH
```

Использовать скрипт `start_gazebo.sh` — он устанавливает пути автоматически.
